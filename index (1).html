HTML создавался и предполагался, как язык, который нет необходимости оформлять. То есть изменение размера текста, фона, добавление рамок и остального не планировалось. По сути своей, разрабатывался он в качестве проектирования информации в более удобном виде, в виде гипертекста. Во времена его создания ещё даже не существовало понятия веб-дизайна и верстки, все сайты по оформлению были буквально одинаковыми. Этот стиль сейчас называется "академическим дизайном".

АКАДЕМИЧЕСКИЙ ДИЗАЙН

Так иногда деликатно именуют отсутствие всякого дизайна. То есть голый белый лист с чёрными буковками. Разумеется, есть и более формальные определения. Например, Д. Кирсанов определяет "академический стиль" как способ оформления документа, при котором используется крайне ограниченный набор тэгов, а именно – так называемые "структурные" теги. Он же пишет:

HTML предназначен именно для разметки логической, не визуальной, структуры, а внешний вид документа в браузере есть лишь побочный эффект логической разметки.

Идеал академического дизайна крайне прост. Это одна html-страница (обычно называемая index.html), содержащая отформатированный текст. Превращать текст в html-файл имеет смысл в том случае, если этот документ будут читать в онлайне, и при том он содержит какое-то интересное оформление.

var para = document.querySelector('p');

para.onclick = function() {
  alert('Owww, stop poking me!');
}
